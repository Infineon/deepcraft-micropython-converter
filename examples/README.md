# DEEPCRAFT Model Usage Examples

This directory contains example scripts demonstrating how to use DEEPCRAFT models in MicroPython applications. These examples show complete workflows from sensor initialization to real-time inference.

## Prerequisites

Before running these examples, ensure you have:

1. **Converted Model**: A `.mpy` model file generated using [DEEPCRAFT MicroPython Converter](../README.md)
2. **Hardware**: Compatible MicroPython device (e.g., Infineon PSOC™ 6 CY8CKIT-062S2-AI)
3. **Sensors**: Appropriate sensors for your model (e.g., BMI270 IMU for motion models)
4. **MicroPython Firmware**: Device running MicroPython with necessary drivers ([BMI270 driver used](https://github.com/Infineon/deepcraft-micropython-data-acquisition/tree/main/sensors))

## Available Examples

### 1. IMU Motion Model (`imu_motion_model.py`)

A complete example demonstrating real-time motion classification using a BMI270 IMU sensor and a DEEPCRAFT model.

#### What This Example Does

- **Sensor Integration**: Initializes and configures the BMI270 IMU sensor (accelerometer + gyroscope)
- **Interrupt-Driven Acquisition**: Uses hardware interrupts for efficient data collection
- **Real-Time Inference**: Classifies motion patterns (e.g., "circle", "shaking") in real-time
- **Sliding Window Processing**: Implements a sliding window approach with automatic stride handling

#### Hardware Setup

**Required Components:**
- Infineon PSOC™ 6 AI Kit with MicroPython firmware flashed
- BMI270 IMU sensor

**Pin Configuration:**
- I2C SCL: `P0_2`
- I2C SDA: `P0_3`
- Interrupt Pin: `P1_5` (data-ready interrupt from BMI270)

**Sensor Configuration:**
- Accelerometer Range: ±8g
- Gyroscope Range: ±500 dps 
- These ranges must match the configuration used during model training in DEEPCRAFT Studio

#### Model Requirements

Your DEEPCRAFT model should be trained with:
- **Input Features**: 6 features per sample (3-axis accelerometer + 3-axis gyroscope)
- **Input Units**: Acceleration in m/s², Angular velocity in dps
- **Sliding Window**: Configure `WINDOW_SIZE` constant to match your model's window size
- **Stride**: The example assumes a stride of 10 samples (configurable in model)

#### Configuration

Before running, update these settings in the script:

```python
# Line 164: Update class labels to match your model output
class_labels = ["unlabelled", "circle", "shaking"]

# Line 13: Update window size to match your model configuration
WINDOW_SIZE = 100
```

**Finding Your Class Labels:**
Open the `model.h` file generated by DEEPCRAFT Studio and look for the `IMAI_DATA_OUT_SYMBOLS` array.

#### Code Structure

The example is organized into modular functions:

1. **`sensor_init()`**: Initializes BMI270 sensor with interrupt configuration
   - Configures I2C communication
   - Sets accelerometer and gyroscope ranges
   - Enables data-ready interrupts

2. **`initialize_model()`**: Creates and initializes the DEEPCRAFT model instance
   - Instantiates the model
   - Retrieves input/output dimensions

3. **`create_buffers()`**: Allocates memory buffers for data exchange
   - Creates `array.array('f')` buffers for efficient C-level memory access
   - Avoids MicroPython object overhead

4. **`run_inference()`**: Main inference loop
   - **Phase 1**: Fills the sliding window with initial samples
   - **Phase 2**: Performs inference on each new sample (respecting stride)
   - Displays predictions with confidence scores

#### Understanding the Inference Workflow

**Phase 1 - Window Filling:**
```
Samples: 1 → 100
Action:  Enqueue only (building initial window)
Output:  None
```

**Phase 2 - Active Inference:**
```
Samples: 101+
Action:  Enqueue new sample + Dequeue (every stride interval)
Output:  Predictions every 10 samples (stride = 10)
```

**Sample Flow:**
```
BMI270 → Interrupt → Read Sensor → Convert Units → Enqueue → Dequeue → Classify
```

#### Key Implementation Details

**Unit Conversion:**
```python
# Accelerometer: g → m/s²
enqueue_buffer[0] = acc_x * GRAVITY_EARTH  # 9.80665 m/s² per g
enqueue_buffer[1] = acc_y * GRAVITY_EARTH
enqueue_buffer[2] = acc_z * GRAVITY_EARTH

# Gyroscope: already in dps (no conversion needed)
enqueue_buffer[3] = gyro_x
enqueue_buffer[4] = gyro_y
enqueue_buffer[5] = gyro_z
```

**Return Codes from `dequeue()`:**
- `0`: Successful inference, output buffer contains class probabilities
- `-1`: Stride not complete, continue collecting samples
- `-2`: Memory allocation error (critical failure)

#### Troubleshooting

**No predictions appearing:**
- Verify `WINDOW_SIZE` matches your model configuration
- Check that sensor is connected and interrupts are firing
- Ensure class labels match model output dimension

**Wrong classifications:**
- Verify sensor ranges (±8g accel, ±500 dps gyro) match training data
- Check unit conversions (m/s² for acceleration)
- Confirm sensor orientation matches training setup

**Memory errors:**
- Reduce buffer sizes if running low on RAM
- Check that model size is compatible with device memory

**Sensor initialization fails:**
- Verify I2C pin connections
- Check sensor power supply
- Test I2C communication with sensor library examples

## General Model Usage Pattern

All DEEPCRAFT models follow this general pattern:

### 1. Initialize Model
```python
import deepcraft_model

model = deepcraft_model.DEEPCRAFT()
model.init()
```

### 2. Get Dimensions
```python
input_dim = model.get_model_input_dim()    # Features per sample
output_dim = model.get_model_output_dim()  # Number of classes
```

### 3. Create Buffers
```python
import array

# Input buffer: sensor data
enqueue_buffer = array.array('f', [0.0] * input_dim)

# Output buffer: class probabilities
output_buffer = array.array('f', [0.0] * output_dim)
```

### 4. Enqueue Data
```python
# Fill buffer with sensor readings
enqueue_buffer[0] = sensor_value_1
enqueue_buffer[1] = sensor_value_2
# ... fill all input_dim values

# Send to model
model.enqueue(enqueue_buffer)
```

### 5. Dequeue Results
```python
result = model.dequeue(output_buffer)

if result == 0:
    # Inference complete
    predicted_class = output_buffer.index(max(output_buffer))
    confidence = max(output_buffer)
elif result == -1:
    # Waiting for more samples (stride not complete)
    pass
elif result == -2:
    # Error occurred
    print("Memory allocation error")
```
